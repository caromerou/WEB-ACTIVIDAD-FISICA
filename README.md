# WEB-ACTIVIDAD-FISICA

Gestión del Proyecto (Agile Workflow)
Este proyecto se gestiona mediante un GitHub Project Board con una estructura personalizada para maximizar la trazabilidad:

🚀 Cómo navegar el Board
Para evitar un tablero infinito, utilizamos Vistas (Views).

Recomendación: No busques columnas por Sprint. Usa las pestañas de View arriba del tablero y aplica el filtro label:"Sprint X" para ver solo el trabajo actual.

🛠️ Estructura de las Tarjetas
Cada historia de usuario se desglosa en:

Tasks DEV: Tareas técnicas de desarrollo.

Tasks QA: Pruebas de calidad y validación.

Bugs: Errores encontrados durante el Sprint.

🔄 Estados (Campos Desplegables)
No movemos tarjetas de columna a columna. El estado de cada tarea se gestiona dentro de la tarjeta mediante menús desplegables para mantener el orden visual:

In Progress | Retesting | Closed | Reopened

2. En la descripción del Proyecto (Dentro del Board)
Para que quien esté viendo el tablero entienda qué hacer sin salir de ahí:

Dentro de tu Project Board, haz clic en el nombre del proyecto arriba a la izquierda (WEB ACTIVIDAD FISICA BOARD).

Se abrirá un menú. Busca la opción que dice "Edit details" o busca el botón "ℹ️ Info" arriba a la derecha.

Ahí verás un cuadro de texto llamado "Description".

Pega ahí un resumen corto: "Board organizado por Sprints. Usar filtros de View para navegar. Los estados se manejan mediante etiquetas y campos internos, no por movimiento de columnas."
